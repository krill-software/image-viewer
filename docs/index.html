Features

Just enough viewer.

← / →

Browse the folder, not a library

Open one image; arrow keys step through siblings in the same directory. No vault, no thumbnails strip, no library sidebar.

Fit · 100% · Free

Three zoom modes

Fits to window by default. Ctrl 1 for actual size, Ctrl scroll to zoom anywhere on the cursor.

Click-drag

Pan when zoomed

Grab and drag the image when it's larger than the window. Subtle scrollbars too.

F

Fullscreen

One key removes the titlebar and status line. Just the image. Esc to return.

PNG · JPEG · WebP · GIF · AVIF · SVG · BMP · ICO

Whatever the webview renders

No heavyweight decoder bindings. Animated GIFs, WebP, and AVIF play on loop automatically.

Tauri · Rust

Native Linux

Small binary. Real file associations. Set as the default for image files via the .deb.

Principles

One window, one image.

No editing. Not even rotate-and-save. That's a different app.
No chrome on the working surface. Filename in the titlebar, dimensions in the status line, image fills the rest.
Fit is always the default. Every new image starts at fit-to-window. Zoom is a deliberate gesture.
Keyboard-first, mouse-honest. Arrow keys browse, Ctrl scroll zooms, click-drag pans.
Install

Two files. One minute.

The .deb registers the app as a candidate handler for image files — right-click any image, "Open with…", set as default.

Or build from source: pnpm install && pnpm tauri build.

Debian / Ubuntu

sudo dpkg -i Image\ Viewer_0.1.0_amd64.deb

AppImage · any Linux

# download
curl -LO https://github.com/krill-software/image-viewer/releases/download/v0.1.0/Image%20Viewer_0.1.0_amd64.AppImage

# run
chmod +x Image\ Viewer_0.1.0_amd64.AppImage
./Image\ Viewer_0.1.0_amd64.AppImage path/to/photo.jpg
